ROC Mondriaan



Hoger-Lager

Dit was mijn aller eerste project op ROC Mondriaan. In dit project werkte ik samen met een goede vriend van mij in een duo en we hadden voor dit project 6 weken met per week 4 uur les. Dit project is gemaakt met Html, CSS en JavaScript en we hebben gewerkt in Visual Studio Code, ook hebben we gebruikt gemaakt van Github ook al werkte we in dit project nog in een persoonlijke repository. Tijdens het projects maakte we gebruik van scrum op papier om goed te kunnen overzien wat we per dag moesten af maken. Tijdens dit project heb ik kunnen leren hoe je goed samen kunt developen en hoe je een goed stappenplan maakt om je projects af te krijgen binnen de deadline. Ook heb ik tijdens dit project leren werken met Github.

Het spel zelf houd in dat je van te voren een gok moet doen, of de worp hoger of lager word dan de worp op het scherm. Als je dit goed heb geraden krijg je een punt, zo niet gaat de score wil naar 0. Wel houd ons programma een highscore bij die nooit terug naar 0 gaat maar wel blijft optellen als je hoger scored dan eerst. Als extra uitdaging hebben wij geprobeert zonder enige uitleg de local database te gebruiken, dit is ons als volgt gelukt: Bij starten van het spel krijg je de optie om een playername in te vullen, als deze naam al eerder is ingevuld in de local database, krijg je de highscore terug die vorige keer door die playernaam is behaald.

Build-a-Game

In dit project werkte ik samen in een groepje van 4 met mijn klasgenoten. Voor dit project hadden 8 weken. Dit project is gemaakt met Html, CSS en JavaScript, we hebben gewerkt in Visual Studio Code en hebben gebruikt gemaakt van Github, Waarin we in een gezaamlijke repository werkte. Wel hadden we in de repository allemaal onze eigen map waarin ons spel zat, alleen de home pagina was gedeeld. Tijdens het projects maakte we gebruik van scrum op papier om goed te kunnen overzien wat we per dag moesten af maken. Tijdens dit project heb ik kunnen leren hoe ik samen werk in een grote groep met mensen die ik daarvoor nog niet echt kende. Ook heb ik veel geleerd over grid.

Project Build a Game is een spelletjes website met (in ons groepje) 4 spellen. Ik zelf heb ganzenbord gemaakt, een typisch Nederlands spel waarbij je met 2 dobbelstenen gooit om precies op vakje 63 te belanden. Dit spel speel je met 2 of meer, in mijn programma tot nu toe alleen met 2. Ook zijn er op het bord veel speciale vakjes te vinden die allemaal een ander effect hebben zoals: je worp verdobbelen, beurten overslaan of zelfs helemaal opnieuw moeten beginnen. Het bord Heb ik gemaakt met grid en aangezien het iconische ganzenbord bord in een spiraal loopt heb ik een spiraal functie moeten maken in JavaScript.

Review-your-Experience

In dit project werkte ik samen met 2 van mijn vrienden in een trio. Voor dit project hadden we 16 weken. Dit project is gemaakt met Html, CSS en PHP, we hebben gewerkt in PhpStorm en hebben gebruikt gemaakt van Github waarin we in een gezaamlijke repository werkte. Tijdens het project maakte we gebruik van de online Github scrum optie, waarin onze groepsleider de taken goed kon verdelen en overzien. Dit project is grotendeels gevult met informatie uit onze database. Tijdens dit project heb ik leren werken met xampp en phpmyadmin. Wel was mijn focus in dit project de Html en CSS, want dat was de taak verdeling.

In project Review your Experience moesten we een website maken waarin we bepaalde producten verkopen. Wij heb gekozen voor een slagerij genaamd Slagerij no SAN, waarin we verschillende soorten vlees verkopen. Zoals eerder gezegt was mijn main focus de Html en CSS, dus ik heb voor alle pagina's de layout, kleurenschema en positionering van de producten die worden geprint uit de database. Tuurlijk werkte we ook samen dus heb ik geholpen met het opzetten wat de database, vooral de relaties leggen tussen de verschillende tabellen. Ook werd ik af en toe geholpen met de CSS en de teksten bedenken die op de About-Us en Home pagina te zien zijn.

Own Projects



Randomizer

Zelf game ik heel veel, maar soms ben ik niet goed ik keuzes maken tijdens het gamen. "Welke character zal ik kiezen?", "Welke map zal ik spelen?", "Links of Rechts?", al dit soort vragen komen voor terwijl ik games speel. Er bestaan al heel wat randomizer websites voor veel spellen en ook bijvoorbeeld een aantal website waarin je zelf een rad kan maken met allerlei opties die hij dan voor je draait, maar dat vind ik zelf vrij saai. Tijdens het spelen van een van mijn favoriete games BTD6 samen met een vriend konden we niet kiezen welke map we wilde spelen, dus ik zocht voor een randomizer website, maar zag dat er voor deze game nog geen was. Toen kwam ik bij het idee er zelf een te maken. De bedoeling van deze website is om in de toekomst meerdere randomizers op een grote plek te hebben, maar voor nu is het alleen voor BTD6. Helemaal af is die ook nog niet, maar de randomizer werkt. Dit project heb ik zelf gemaakt in 1 week tijd. Ik heb gebruikt gemaakt van Html, CSS en JavaScript, misschien dat ik in de toekomst gebruik maak van PHP met een database. Tijdens dit project heb ik geleerd dat ik coderen echt leuk vind en dat ik het niet alleen voor school hoef te gebruiken.

Op de website heb je voor nu 3 dingen die je kunt randomizen voor de game, als je die game niet kent is het niet raar als je de termen niet kent van de randomizer. Je kunt de map die je wilt spelen randomizen, ook per difficuly, dus als je bijv. alleen expert maps wil kan dat. Ook kun je je Hero randomizen, bij deze heb ik die optie gegeven om met skins te randomizen of alleen de defaults, of allebei, omdat je in de nieuwe gamemode (Legends) de skin van de Hero ook uitmaakt. Als laatst kun je de Tower die je wilt plaatsen randomizen, ik wil nog gaan toevoegen dat je per tower ook de path erbij kunt randomizen, maar voor nu is het alleen de base Tower. Wel kun je kiezen wat voor soort Tower je wilt randomizer bijv. alleen primary of magic. Ooit wil ik dit een echte website maken, voor nu kan ik hem alleen gebruiken via Visual Studio Code.
